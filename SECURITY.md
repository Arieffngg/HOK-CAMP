# 🛡️ Kebijakan Keamanan (Security Policy)

Terima kasih atas minat Anda terhadap proyek **HOK Camp Intelligence Engine**. Sebagai pengembang, kami menganggap serius keamanan data dan integritas sistem.

## 📌 Versi yang Didukung (Supported Versions)

Saat ini, hanya versi mayor terbaru dari arsitektur ini yang secara aktif menerima pembaruan keamanan (*Security Patches*).

| Versi Sistem | Status Dukungan Keamanan |
| :--- | :--- |
| **V4.x** (Current) | ✅ Didukung Secara Aktif |
| V3.x & ke bawah | ❌ Tidak Didukung (End of Life) |

## 🚨 Cara Melaporkan Kerentanan (Reporting a Vulnerability)

Jika Anda menemukan celah keamanan (*vulnerability*), kelemahan logika *scraper*, atau potensi kebocoran API di dalam *source code* ini, kami memohon kerja sama Anda sebagai sesama *developer*.

**TOLONG JANGAN DIBUKA KE PUBLIK:**
Jangan membuat *Public Issue* atau *Pull Request* terbuka di GitHub mengenai celah tersebut, karena dapat dieksploitasi oleh pihak yang tidak bertanggung jawab sebelum kami sempat menambalnya.

**Langkah Pelaporan (Responsible Disclosure):**
1. Harap laporkan temuan Anda secara rahasia dengan mengirimkan pesan langsung (DM) kepada *Tech Lead / Maintainer* utama (Arief).
2. Anda juga dapat menggunakan fitur **Private Vulnerability Reporting** di GitHub jika opsi tersebut telah kami aktifkan di tab *Security*.
3. Sertakan bukti konsep (*Proof of Concept* / PoC) atau langkah-langkah detail untuk mereproduksi masalah tersebut.

Kami akan melakukan investigasi mendalam, merilis *patch* keamanan sesegera mungkin, dan tentu saja memberikan **kredit/apresiasi** kepada Anda dalam *Release Notes* kami. 

Terima kasih telah membantu menjaga ekosistem komunitas HOK tetap aman dan profesional!
